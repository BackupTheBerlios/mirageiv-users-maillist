Pyexiv2 is added as an optional dependency in the svn version to get
exif data from jpg images. That ( and possibly together with a module
for jpegtran(I have in a private repo)) will make it possible to do
auto-rotation (losslessly). What is required is to patch up mirage to
actually do this. This is actually one of the feature requests on the
tracker at berlios and one I would like to add. Unfortunately I don't
know when I will get around to code it.

If you can program in python that understand mirage's code I would be
very glad if you or someone else is able to code this up.
