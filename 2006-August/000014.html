Hi,

I patched mirage.py, so that you can define keyborard
shortcuts
on the command line:

There are two new command parameter:
-c, --command        'k: mycommand IMAGE' Execute  
                    'mycommand' if you hit key 'k'"
                   'k' can be any ASCII character. 
                   You can give several -c options
                  IMAGE gets replaced with the current
                   filename
                 If the command contains the string
                 NEXT, the next 
                image gets loaded after the command

-o, --onload         'mycommand IMAGE' 
        Execute 'mycommand',when the image gets loaded

This way you can easily sort a lot of pictures into
different directories:

Example:

mirage --onload "echo IMAGE" \
    --command 'g: mv IMAGE good-dir NEXT' \
    --command 'b: mv IMAGE bad-dir NEXT' \
    --command 't: mv IMAGE trash-dir NEXT'  files....

Just hit g, b or t now to sort the images into
different directories.

"echo IMAGE" is quite useless in the example, but you can
do other things: Print Exif info, print OCR results, ...

I tried to add the patch to the berlios page as feature
request or as bug report, but a file upload was not
possible.

mirage.py contains space and tabs mixed. I normalized
the code to four spaces. The patch only contains the
important changes, but indentation might be wrong.
Most Python software uses four spaces. I use
emacs to normalize it:

set-variable tab-width 8
mark-whole-buffer
tabify
set-variable tab-with 4
untabify
