I've commited to svn a good chunk of code that should accomodate what
Thomas wanted, and more. First I implemented Custom Actions, where you
can create your own actions to do things like generating thumbnails or
other manipulations via external apps, moving image to other dirs, and
so on.  Each custom action gets its own shortcut and is saved in the
config, so there is no need to have commandline arguments as well.
(Additionally, commandline arguments could present shortcut
collisions.) Custom actions can also be applied to all images in the
list, so that you can load a directory of images, for example, and
create thumbnails for all of them.  Mirage will also update any images
if they are modified by the custom action.

I've also implemented your --onload, with the slight modification that
the same syntax available for custom actions is also used for this.
The full list is available from Edit > Custom Actions.

Let me know if anyone runs into problems, wants additional
properties/operations available, etc.
